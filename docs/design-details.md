# 设计细节文档

> 本文档是对 [gameplay.md](gameplay.md) 的补充，明确了具体的设计决策和实现细节，避免重复推理。

---

## 1. 核心设计哲学

**一切通过标签实现，保证涌现性。**

标签（Tag）是本游戏最底层的机制单元。硬币的行为、属性、能力、状态——全部由标签组合定义。这意味着：
- 设计新硬币 = 组合一组标签
- 设计 Boss 盲注 = 在场上生成带有特定标签的硬币
- 设计新流派 = 围绕某类标签构建策略
- 涌现性：当两个标签组合产生设计者未预料到的效果时，这是特性而非 bug

---

## 2. 标签系统（Tag System）

### 标签是什么
标签是附着在硬币上的**行为/属性修饰符**。每个标签定义一个独立的规则或效果，多个标签组合在一起定义了一个硬币的完整行为。

### 标签分类

#### 按稀有度
| 稀有度 | 说明 |
|--------|------|
| **常见** | 基础标签，容易获得，效果温和 |
| **罕见** | 稀有标签，效果强力或独特 |

#### 按正负性
| 类型 | 说明 |
|------|------|
| **正面标签** | 提供增益效果 |
| **负面标签** | 提供减益/限制效果，通常出现在 Boss 盲注硬币或作为强力能力的代价 |

### 标签附着规则
- 每个硬币根据其**品质/稀有度**决定最多可附着的标签数量：
  - 普通品质：最多 **1** 个标签
  - 稀有品质：最多 **2** 个标签
  - 传说品质：最多 **3** 个标签
- 上述限制仅针对**玩家主动附着**的标签。硬币自带的固有标签不受此限制。

### 标签示例

#### 正面标签（常见）
| 标签名 | 效果 |
|--------|------|
| **回归** | 掉出场地时，会重新回到场地 |
| **再弹跳** | 落地后会再弹跳一次 |
| **正面翻倍** | 正面朝上时，该硬币得分翻倍；背面朝上时不拿分 |

#### 正面标签（罕见）
| 标签名 | 效果 |
|--------|------|
| **磁力** | 落地后吸引周围可被吸引的硬币 |
| **兔子** | 落地后，在场地上增加 x 只小兔子硬币（x = 本次翻转次数） |

#### 负面标签
| 标签名 | 效果 |
|--------|------|
| **虚无** | 游戏结束后（本局结束后）消失 |
| **脆弱** | 翻转后消失 |
| **超重** | 该硬币质量 ×10 |
| **尸体** | 该硬币面值归零 |

#### 特殊标签
| 标签名 | 效果 |
|--------|------|
| **生物** | 会被砸死；被砸死后变为尸体（移除【生物】标签，添加【尸体】标签） |

### 标签的涌现性示例

**小兔子硬币**（面值 1）由以下标签组合定义：
- 虚无：游戏结束后消失
- 脆弱：翻转后消失
- 生物：会被砸死，砸死后变为尸体

**小兔子尸体硬币**（小兔子被砸死后）：
- 虚无：游戏结束后消失
- 脆弱：翻转后消失
- 尸体：面值归零

**涌现可能性**：因为"生物被砸死变尸体"是一个通用机制，未来可以设计：
- "亡灵"流派：围绕尸体标签构建策略（如"场上每有一个尸体硬币，得分 +5"）
- "牧场"流派：围绕生物标签构建策略（如"场上每有一个生物硬币，倍率 +0.5"）
- 任何带有【生物】标签的硬币都自动获得被砸死→变尸体的行为，无需额外编码

## 3. 得分系统（双乘区）

### 核心公式

```
最终得分 = 基础得分（Chips） × 倍率（Mult）
```

类似 Balatro 的 Chips × Mult 双乘区设计。

### 基础得分（Chips）
- 场上所有硬币的**面值之和**。
- 立起来的硬币额外 +10 Chips。
- 标签、遗物等可以增加 Chips。

### 倍率（Mult）
- 初始倍率为 **1.0**。
- 通过标签、遗物、流派等增加倍率。
- 倍率的增加方式：
  - **加法**（+Mult）：如"正面朝上的硬币 +0.5 倍率"
  - **乘法**（×Mult）：如"正面翻倍标签：该硬币得分部分 ×2"
  - 先计算所有加法，再计算所有乘法。

### 结算流程
1. 所有硬币静止后，逐个硬币计算贡献：
   - 该硬币的 Chips 贡献（面值 + 标签/能力加成）
   - 该硬币的 Mult 贡献（标签/能力提供的倍率加成）
2. 汇总所有 Chips，汇总所有 Mult
3. 最终得分 = 总 Chips × 总 Mult
4. 播放逐个硬币计分的演出动画（类似 Balatro）

### 回合结束奖励
- 根据剩余翻转次数获得金币。
- 获得的金币以硬币形式掉落在场地上（成为下一局的硬币）。

## 4. 视角与摄像机

- **固定略倾斜俯视角**，俯角约 60°-70°，能同时看到硬币正面和侧面厚度。
- 摄像机**不可旋转**，保持稳定的空间映射关系，确保点击精度。
- 像素风格纹理在固定视角下不会因旋转失真。

---

## 5. 点击交互机制

### 翻转操作
- **单击触发**，无蓄力/拖拽过程。
- 点击点相对于硬币中心的偏移向量决定力矩方向：
  - 点击左边 → 力矩使硬币向右翻转。
  - 点击右边 → 力矩使硬币向左翻转。
  - 点击中心附近 → 主要是垂直弹起，随机轻微翻转。
  - 点击越靠近边缘，力矩越大，翻转越剧烈。
- 力的大小 = `基础力度 ± 随机浮动`，"精准"属性缩小浮动范围。
- **鼠标移动速度影响翻转方向**：点击瞬间的鼠标移动速度会转换为水平方向的力分量，类似打砖块中碰撞时移动方向影响弹射方向。快速划过硬币点击会让硬币朝鼠标移动方向偏移飞出。

### 翻转次数消耗
- **一次点击 = 消耗一次翻转机会**，不管物理连锁带动了多少硬币。
- 翻转下面的硬币带动上面的硬币是免费的物理连锁奖励，这是策略的一部分。

## 6. 结算机制

### 结算时机
- 玩家用完所有翻转次数（或主动结束回合）后，等待所有硬币静止。
- **静止判定**：所有 RigidBody3D 的线速度和角速度均低于阈值时，视为全部静止，触发统一结算。

### 正反面判定
- 硬币的 up 向量与世界 up 向量的点积：
  - `> 0` → 正面朝上
  - `< 0` → 反面朝上

### 立起来判定
- 硬币 up 向量与桌面法线（世界 up）的夹角 > 约 70° 时视为"立起来"。
- 容差约 ±15°-20°，不要求严格 90°（物理模拟下几乎不可能稳定在 90°）。
- 立起来的硬币额外 +10 Chips。

### 翻转次数计数（物理翻转）
- 硬币从被弹起到落地静止期间，监测其**绕水平轴的累计旋转角度**。
- 每完整旋转 360° 计为一次翻转。
- 用于"兔子"等按翻转次数触发效果的标签。

## 7. 场地与边界

- 场地是一个有边界的桌面/平台。
- **硬币默认可以掉出场地** → 掉落的硬币丢失。
- 这是一个惩罚机制，增加翻转操作的风险收益权衡。
- 让"精准"属性更有价值——力度浮动大时更容易把硬币弹飞出场。
- **【回归】标签**可以让硬币掉出场地后自动回到场地。

---

## 8. 商店与支付机制

### 支付流程
- 进入商店后，场地上的硬币仍然可见。
- 玩家点击硬币将其"弹起" → 硬币飞走，视为支付（不会再落下）。
- 硬币的面值 = 它作为货币的价值。
- 持续点击支付，直到累计支付金额 ≥ 商品价格。

### 找零机制
- 超额支付时，商店"找零"：多出的金额以小面值硬币的形式从上方掉落到场地上。
- 找零硬币掉落时可能堆叠在现有硬币上。

### 核心决策张力
- 硬币同时是**得分工具**和**货币**。
- 花掉高面值/高能力的硬币会削弱得分能力。
- 花掉低面值硬币可能需要更多次支付操作。

## 9. 硬币物理属性差异化

| 属性 | 大硬币 | 小硬币 |
|------|--------|--------|
| 点击难度 | 容易 | 困难 |
| 重量 | 重，不易被弹飞 | 轻，容易被碰飞 |
| 堆叠 | 容易挡住下面的硬币 | 容易堆叠在其他硬币上 |
| 遮挡 | 可能遮挡下层硬币的点击 | 不太会遮挡 |

- 摩擦力、弹性等物理属性可调节，影响翻转和碰撞效果。
- 物理属性差异是构筑策略的一部分——选择硬币组合时不仅看能力，还要考虑物理交互。
- **【超重】标签**可以将硬币质量 ×10，极大改变物理交互。

## 10. 游戏流程与局制结构

### 层级关系（对标 Balatro）

```
一轮 Run（一次完整游戏）
├── Ante 1（第一幕）
│   ├── Round 1（小盲注）→ 翻转阶段 → 结算 → 商店
│   ├── Round 2（大盲注）→ 翻转阶段 → 结算 → 商店
│   └── Round 3（Boss 盲注）→ 翻转阶段 → 结算 → 商店
├── Ante 2（第二幕，目标分数递增）
│   └── ...
└── Ante 8（最终幕）
```

- **Round（局）**：一次翻转 + 结算的完整循环。玩家有有限翻转次数，用完或主动结束后结算。
- **Ante（幕）**：包含若干局，每幕有递增的目标分数。
- **Run（轮）**：一次完整的游戏流程，从 Ante 1 到通关或失败。

### Boss 盲注机制
Boss 盲注通过标签系统自然实现——在场上生成带有特定标签组合的特殊硬币，改变本局的物理环境和策略。

**示例：【黑曜石】盲注**
- 在场上随机位置生成 10 个黑曜石硬币。
- 黑曜石硬币（面值 10）：
  - **虚无**：游戏结束后消失
  - **超重**：质量 ×10
- 效果：超重的黑曜石硬币会压住其他硬币使其翻不起来，还可能把硬币挤出场地。
- 策略：玩家需要想办法先翻开黑曜石（或利用黑曜石的高面值得分）。

**更多 Boss 盲注方向**（待设计）：
- 生成大量【脆弱】硬币干扰场地
- 生成带【磁力】的硬币打乱布局
- 给玩家现有硬币附加负面标签

### 目标分数与失败条件
- 每局有一个**目标分数**，最终得分（Chips × Mult）≥ 目标即为通过。
- 目标分数随 Ante 递增（具体数值待平衡）。
- **未达到目标分数 = Game Over**，本轮 Run 结束。

### 局与局之间的继承
- 场上的硬币**全部继承**到下一局（包括位置和状态）。
- 带有【虚无】标签的硬币在局结束后消失，不继承。
- 回合结束奖励的金币掉落后，也成为下一局的初始硬币。
- 在商店中被支付掉的硬币从场地移除，不再继承。

### 跳过机制
- 玩家可以选择**跳过战斗（翻转回合）和商店环节**。
- 跳过后获得**特殊事件奖励**作为补偿，例如：
  - "复制一个金币"（在场上生成一个相同硬币的副本）
  - "将 A 硬币的标签移动到 B 硬币上"
  - "免费获得一个遗物硬币"
  - "所有硬币品质提升一级"
- 跳过是一种**风险收益权衡**：放弃本局得分和商店购买机会，换取可能更有价值的特殊奖励。

## 11. 道具与遗物系统

### 道具（Items）
- **一次性使用**，使用后从背包中移除。
- 最多同时持有 **3 个**，超出时必须丢弃或使用。
- 通过商店购买或特殊事件获得。
- 效果示例：
  - "本局额外 +2 翻转次数"
  - "选择一个硬币，将其固定在当前位置（本局不受物理影响）"
  - "将场上所有硬币翻到正面"

### 遗物（Relics）
- **全局持久效果**，没有数量上限。
- 遗物的载体是**场上的特殊硬币**——遗物硬币同时参与物理模拟和得分。
- 只要遗物硬币在场上，其全局效果就持续生效。
- 如果遗物硬币被支付掉或掉出场地（且没有【回归】标签），对应的全局效果消失。
- 效果示例：
  - "所有硬币面值 +1 Chips"
  - "每局额外 +1 翻转次数"
  - "立起来的硬币额外 +5 Chips"
  - "找零时额外获得 1 枚铜币"
- 遗物硬币本身也可以拥有标签，标签和遗物效果共存。

## 12. 硬币面值与品质体系

### 基础面值
| 硬币类型 | 面值 | 大小 | 品质 |
|---------|------|------|------|
| 铜币 | 1 | 小 | 普通（最多附着 1 个标签） |
| 银币 | 5 | 中 | 稀有（最多附着 2 个标签） |
| 金币 | 10 | 中大 | 传说（最多附着 3 个标签） |
| 大金币 | 25 | 大 | 传说（最多附着 3 个标签） |

> 具体面值、种类和品质对应关系待平衡调整，以上为初始设计。
> 品质限制的是**玩家主动附着**的标签数量，硬币自带的固有标签不受此限制。

### 品质系统
- 品质决定硬币可附着的**标签数量上限**（普通 1 / 稀有 2 / 传说 3）。
- 品质可通过特殊事件提升（如"所有硬币品质提升一级"）。
- 品质还可能影响硬币的视觉表现（边框颜色、光效等）。

### 强化
- **永久提升**硬币的基础属性（如面值 +1、重量 -10%）。
- 可叠加多次，通过商店或随机事件获得。

## 13. 硬币定义示例（标签组合）

以下示例展示如何通过标签组合定义不同的硬币：

### 小兔子硬币（由【兔子】标签生成）
| 属性 | 值 |
|------|-----|
| 面值 | 1 |
| 大小 | 极小 |
| 固有标签 | 虚无、脆弱、生物 |

行为：
- 局结束后消失（虚无）
- 被翻转后消失（脆弱）
- 被其他硬币砸中后变为尸体（生物）

### 小兔子尸体硬币（小兔子被砸死后）
| 属性 | 值 |
|------|-----|
| 面值 | 1 → 0（被【尸体】标签归零） |
| 大小 | 极小 |
| 固有标签 | 虚无、脆弱、尸体 |

行为：
- 局结束后消失（虚无）
- 被翻转后消失（脆弱）
- 面值归零（尸体）

### 黑曜石硬币（Boss 盲注生成）
| 属性 | 值 |
|------|-----|
| 面值 | 10 |
| 大小 | 中 |
| 固有标签 | 虚无、超重 |

行为：
- 局结束后消失（虚无）
- 质量 ×10（超重），会压住其他硬币，可能挤出场地

### 标签驱动的状态转换

```
[小兔子硬币]  ──被砸死──→  [小兔子尸体硬币]
  虚无                        虚无
  脆弱                        脆弱
  生物  ──移除──→              尸体  ←──添加──
```

**【生物】标签的通用规则**：任何带有【生物】标签的硬币被砸死时，移除【生物】标签，添加【尸体】标签。这是一个通用机制，适用于所有生物类硬币。

## 14. 开场流派系统

### 概念
- 类似 Balatro 的筹码选择 / 杀戮尖塔的角色选择。
- 玩家在游戏开始时选择一个**开场流派**，定制初始游戏体验。
- 解决"构筑成型前玩法固定"的问题，让玩家从第一局就有差异化体验。

### 流派方向（待设计）
- **堆叠流派**：堆叠硬币提供额外 Mult 加成。
- **翻转流派**：额外翻转次数，翻转相关的 Chips/Mult 加成。
- **精准流派**：力度浮动更小，边缘点击加成。
- **赌博流派**：正反面结果的收益/惩罚更极端。
- **亡灵流派**：围绕【尸体】标签构建策略。
- **牧场流派**：围绕【生物】标签构建策略。

---

## 15. 翻转次数与力度成长

### 翻转次数
- **初始翻转次数**：每局 3 次（受开场流派影响）。
- 增加途径：遗物效果、道具、硬币能力、商店购买永久升级。

### 力度成长
- **基础力度**：决定硬币弹起的基础高度和旋转速度。
- 成长途径：
  - 商店中购买"力度升级"（永久 +N%）。
  - 特定遗物效果提供力度加成。
  - 开场流派可能影响初始力度。
- **精准属性**：减少力度的随机浮动范围，同样可通过商店/遗物/流派提升。

## 16. 随机事件系统

- 在特定时机触发（如：通过某一局后、进入商店时、Ante 之间、跳过时）。
- 事件类型：
  - **强化事件**：选择一个硬币进行免费强化。
  - **标签事件**：为一个硬币附着标签。
  - **交换事件**：用一个硬币换取另一个。
  - **挑战事件**：额外的限制条件 + 额外奖励。
  - **复制事件**：复制一个金币。
  - **转移事件**：将 A 的标签移动到 B 上。
- 具体事件池待设计。

## 17. UI 布局（初步规划）

```
┌─────────────────────────────────────────┐
│  [目标: 300]  [Chips: 0 × Mult: 1.0]   │  ← 顶部信息栏（双乘区显示）
├─────────────────────────────────────────┤
│                                         │
│                                         │
│           ┌───────────────┐             │
│           │               │             │
│           │   场地/桌面    │             │
│           │  (硬币+遗物)   │             │
│           │               │             │
│           └───────────────┘             │
│                                         │
├─────────────────────────────────────────┤
│  [翻转次数: 3/3]  [结束回合] [跳过]     │  ← 底部操作栏
│  [道具1][道具2][道具3]                  │  ← 道具槽（最多3个）
└─────────────────────────────────────────┘
```

- 遗物硬币直接在场地上以特殊视觉样式显示（发光边框/特殊材质），不需要单独 UI 槽。
- 硬币悬停时显示信息浮窗：面值、品质、标签列表（含固有标签和附着标签）、遗物效果（如有）。
- 结算时播放逐个硬币计分的动画，分别显示 Chips 和 Mult 的贡献。
- 道具槽支持点击使用，使用时可能需要选择目标硬币。

## 18. 技术选型

- **引擎**：Godot 4.6
- **物理引擎**：Jolt Physics（已在 project.godot 中配置）
- **开发语言**：C#（.NET）
- **渲染**：GL Compatibility
- **核心架构**：标签系统（Tag System）—— 一切行为通过标签组合实现
- **硬币定义**：数据驱动，硬币 = 基础属性 + 标签列表
- **标签实现**：每个标签是一个独立的组件/策略，通过接口定义触发时机和效果
