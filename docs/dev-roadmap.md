# 开发路线图

> 按里程碑划分，每个里程碑是一个可独立验证的交付物。
> 核心原则：**标签系统是一切的基础**，尽早搭建。

---

## Milestone 0：项目骨架（预计 1-2 天）

- [ ] 创建基础目录结构（`scenes/`, `scripts/`, `resources/`, `assets/`）
- [ ] 搭建主场景（Main.tscn）：摄像机、灯光、桌面
- [ ] 配置固定俯视角摄像机（60°-70° 俯角）
- [ ] 创建基础桌面（StaticBody3D + 碰撞体 + 边界墙）
- [ ] 验证 Jolt Physics 正常工作

**验收标准**：打开场景能看到一个有边界的 3D 桌面，摄像机视角正确。

---

## Milestone 1：核心物理原型（预计 3-5 天）

- [ ] 创建硬币场景（RigidBody3D + MeshInstance3D + CollisionShape3D）
- [ ] 实现射线检测点击硬币
- [ ] 实现点击位置 → 力矩方向的映射
- [ ] 实现力度随机浮动（基础力度 ± 浮动范围）
- [ ] 实现硬币静止检测（线速度 + 角速度阈值）
- [ ] 实现正反面判定（up 向量点积）
- [ ] 实现立起来判定（角度阈值）
- [ ] 实现物理翻转次数计数（累计旋转角度 / 360°）
- [ ] 调试硬币物理手感（摩擦力、弹性、质量）

**验收标准**：场上有几个硬币，点击能翻转，落地后能判定正反面和是否立起来，物理手感舒适。

## Milestone 2：标签系统核心（预计 4-6 天）

> 标签系统是一切的基础，必须在游戏循环之前搭建。

### 标签框架
- [ ] 设计 ITag 接口（触发时机、效果执行、标签元数据）
- [ ] 实现标签管理器（附着、移除、查询）
- [ ] 实现标签触发时机框架：
  - 翻转时（OnFlip）
  - 落地时（OnLand）
  - 结算时（OnScore）
  - 被砸时（OnHit）
  - 掉出场地时（OnFallOff）
  - 持续性（Passive）
- [ ] 实现硬币品质系统（普通/稀有/传说 → 标签槽 1/2/3）

### 基础标签实现
- [ ] 虚无（局结束后消失）
- [ ] 脆弱（翻转后消失）
- [ ] 生物（被砸死 → 移除生物标签 + 添加尸体标签）
- [ ] 尸体（面值归零）
- [ ] 超重（质量 ×10）
- [ ] 回归（掉出场地后回到场地）
- [ ] 再弹跳（落地后再弹跳一次）
- [ ] 正面翻倍（正面朝上得分翻倍，背面不拿分）
- [ ] 磁力（落地后吸引周围硬币）
- [ ] 兔子（落地后生成 x 只小兔子硬币）

### 数据驱动硬币定义
- [ ] 硬币定义 = 基础属性（面值、大小、品质）+ 固有标签列表
- [ ] 实现硬币工厂（根据定义生成硬币实例）
- [ ] 定义小兔子硬币、小兔子尸体硬币作为验证

**验收标准**：标签可附着/移除，各触发时机正确触发，小兔子硬币的完整生命周期（生成→被砸死→变尸体）可正常运行。

- [ ] 实现翻转次数系统（初始 3 次，用完后禁止点击）

**验收标准**：能完整玩一个"翻转 → 结算 → 下一局"的循环，双乘区得分正确，有目标判定。

## 技术注意事项

- **语言**：C#，所有脚本放在 `scripts/` 目录下。
- **场景组织**：每个独立功能一个场景（硬币、桌面、UI、商店等）。
- **标签系统**：核心架构。每个标签是一个实现 ITag 接口的独立类，通过组合定义硬币行为。
- **硬币定义**：数据驱动，使用 Resource 或 JSON 定义硬币模板（基础属性 + 固有标签列表）。
- **状态管理**：回合状态机管理游戏流程（翻转阶段 / 等待静止 / 结算 / 商店）。
- **物理调参**：硬币的物理属性（摩擦力、弹性、质量）需要大量调试，建议做一个调试面板方便实时调整。
- **得分系统**：双乘区（Chips × Mult），结算时分别计算并演出。
